# 合规审查清单

发布 AI API、Token、模型调用服务相关内容前，建议按此清单自查。

## 1. 禁止出现或谨慎处理的表达

| 检查项 | 是否出现 | 处理建议 |
|---|---|---|
| 代充、充值、余额、余额池 |  | 删除，改为采购、账单、预算管理 |
| 共享账号、共享 key |  | 删除，改为客户自有 key、权限隔离 |
| 中转站、反向代理转售 |  | 删除，改为企业模型网关和模型路由，并确保业务实质合规 |
| 逆向、破解、绕限制 |  | 删除 |
| 低价倍率、稳定通道 |  | 改为总拥有成本测算、SLA、错误率监控 |
| 测试账号、有需要联系 |  | 改为官方试点、采购评估流程 |
| 普通人卖 token、兼职副业 |  | 改为中小团队 API 采购入门 |
| 收益多少倍、稳赚、轻松入局 |  | 删除，避免高承诺 |
| 私信拿资源、群内发通道 |  | 改为领取合规资料、预约诊断 |

## 2. 合规内容方向

建议优先发布：

- API 采购流程。
- 供应商尽调。
- SLA 和错误率监控。
- 数据处理和授权边界。
- BYOK 网关。
- 客户自有 key 管理。
- 模型路由。
- 成本测算。
- 预算控制。
- 用量审计。
- 权限分级。
- 异常调用告警。

## 3. 发布前自查问题

1. 这条内容是否可能被理解成代充、转售、共享 key 或规避限制？
2. 标题是否过度承诺收益、低价、稳定或无限量？
3. 结尾是否把用户导向资源交易，而不是资料、诊断或采购评估？
4. 是否明确强调官方、授权、客户自有 key、透明账单和审计？
5. 是否避免展示账号、key、充值、余额、可疑接口或购买路径？
6. 是否需要法务、平台规则或供应商条款负责人复核？

## 4. 安全承接清单

可以承接：

- API 采购风险自查表。
- 供应商尽调表。
- BYOK 接入检查表。
- 模型路由方案模板。
- 调用成本测算表。
- 用量审计模板。
- SLA 对比表。
- 企业 API 治理诊断。

不应承接：

- 账号、key、余额、通道、充值。
- 低价倍率表。
- 未授权接口。
- 规避限制教程。
- 资源群、测试账号、暗号私信。

## 5. 最终判断

发布前用一句话判断：

> 这条内容是在帮企业建立可授权、可审计、可治理的 AI API 使用能力，还是在引导用户寻找更便宜、更隐蔽、更不可追溯的资源？

如果答案偏向后者，不建议发布。
