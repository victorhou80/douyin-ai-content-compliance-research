# 03-选题脚本库

这些脚本只用于合规企业 API 采购、BYOK 网关、模型路由、成本治理和用量审计内容，不用于中转、代充、共享 key、余额池、反向代理转售或规避限制。

## 1. API 账单为什么突然失控？

开头：

> 很多团队第一个月 API 账单还能接受，第三个月突然翻倍，不一定是模型贵，而是调用治理没做。

正文：

- 任务没有分级，简单任务也打到高成本模型。
- 没有预算阈值，异常调用没人发现。
- 没有部门和项目维度账单，成本归属不清。
- 没有失败率和重复调用监控。

结尾：

> 先别急着换供应商，先做一张调用成本测算表。

承接资料：

- API 成本检查表。
- 调用成本测算表。

## 2. BYOK 网关到底解决什么？

开头：

> BYOK 不是低价技巧，它解决的是企业自有 key 怎么安全、可控、可审计地使用。

正文：

- 谁可以调用。
- 每个人能调用多少。
- 调用了哪个模型。
- 成本算到哪个项目。
- 出问题能不能追溯。

结尾：

> 如果你的团队已经多人共用 key，就该先做权限和日志治理。

承接资料：

- BYOK 接入自查表。
- 权限分级模板。

## 3. 企业采购 API，最容易踩的 5 个坑

开头：

> API 采购不是价格越低越好，真正贵的是不可控。

正文：

- 授权来源不清。
- 账单不透明。
- SLA 没写清。
- 数据处理边界不明确。
- 退出和迁移机制缺失。

结尾：

> 采购前先用供应商尽调表筛一遍。

承接资料：

- API 采购风险自查表。
- 供应商尽调表。

## 4. 模型路由不是技术炫技，是成本治理

开头：

> 不是所有任务都应该用最强模型。客服、摘要、分类、代码、复杂推理，适合的模型不一样。

正文：

- 简单任务用轻量模型。
- 高价值任务用强模型。
- 失败后再升级模型。
- 对高频任务做缓存和批处理。
- 用路由规则控制预算上限。

结尾：

> 模型路由的目标不是只省钱，而是在成本、质量和稳定性之间做可审计的选择。

承接资料：

- 模型路由方案模板。
- 任务分级表。

## 5. 老板看不懂 API 账单怎么办？

开头：

> 老板不关心你接了多少模型，他关心这笔钱花在哪、带来了什么结果、有没有失控风险。

正文：

- 按部门拆账。
- 按项目拆账。
- 按模型拆账。
- 按用户拆账。
- 标记异常峰值和失败率。

结尾：

> 一张用量审计表，比一句“我们在用 AI”更有说服力。

承接资料：

- 用量审计字段模板。
- 成本复盘表。

## 6. 低价方案为什么最后可能更贵？

开头：

> 低价只是采购成本的一部分，总拥有成本还包括停机、迁移、审计、追责和数据风险。

正文：

- 账单不可追溯。
- 服务不稳定。
- 数据责任不清。
- 后续迁移成本高。
- 企业内部无法通过法务和财务审核。

结尾：

> 企业采购要看总拥有成本，不是只看单次调用价格。

承接资料：

- 总拥有成本测算表。
- SLA 对比表。

## 7. 从个人试用到企业采购，中间差什么？

开头：

> 个人能用，不代表企业能采购。企业要过法务、财务、安全、IT 和业务验收。

正文：

- 合同主体。
- 发票和账单。
- 数据处理条款。
- 权限管理。
- 日志审计。
- SLA 和支持响应。

结尾：

> 试用跑通只是第一步，采购能落地才是真正开始。

承接资料：

- 企业接入流程图。
- 采购评估问卷。

## 8. AI 服务商别卖通道，要卖治理能力

开头：

> 卖通道很容易卷价格，卖治理能力才有长期价值。

正文：

- API 采购评估。
- BYOK 网关实施。
- 模型路由设计。
- 成本预算和告警。
- 用量审计和报表。
- 团队培训和流程建设。

结尾：

> 真正有价值的是帮客户把 AI 调用管起来。

承接资料：

- 服务包结构图。
- 企业 API 治理方案样例。

## 9. API 供应商靠谱不靠谱，看这张表

开头：

> 不要只问能不能用，也要问出了问题谁负责。

正文：

- 授权来源。
- 数据处理边界。
- SLA。
- 账单透明度。
- 支持响应。
- 退出机制。

结尾：

> 供应商不是越便宜越好，而是越可控越好。

承接资料：

- 供应商评分表。
- 采购问题清单。

## 10. 合规 AI 网关应该有哪些基础能力？

开头：

> 网关不是简单转发接口，企业要的是治理层。

正文：

- BYOK。
- 权限分级。
- 模型路由。
- 预算限额。
- 调用日志。
- 异常告警。
- 成本报表。

结尾：

> 能调用只是基础，能管理、能审计、能复盘才是企业级。

承接资料：

- 网关能力清单。
- API 治理成熟度评估表。
