# 02-合规改写手册

## 1. 改写原则

合规改写不是把敏感词换个说法继续做原来的事，而是把业务实质切换到官方、授权、透明、可审计、可治理的企业服务方向。

正确方向：

- API 采购评估。
- 官方或授权供应商接入。
- 客户自有 key 的 BYOK 管理。
- 多模型路由。
- 预算控制和成本治理。
- 权限分级和调用日志。
- 用量审计和异常预警。
- 供应商尽调、SLA、数据处理和退出机制评估。

不应提供或暗示：

- 中转、代充、共享账号、共享 key。
- 余额池、反向代理转售。
- 逆向接口、规避地区限制或平台限制。
- 未授权低价通道。
- 测试账号或账号/key 分发。

## 2. 高风险词替换

| 高风险表达 | 风险原因 | 推荐改写 |
|---|---|---|
| 卖 token | 容易指向资源转售 | API 成本治理、模型调用服务 |
| API 充值 | 容易指向代充或余额池 | API 采购、账单管理、预算控制 |
| 中转站 | 容易指向反向代理转售 | 企业模型网关、模型路由、统一调用入口 |
| 低价倍率 | 容易变成价格套利 | 总拥有成本测算、预算优化、成本结构分析 |
| 测试账号 | 容易涉及账号分发 | 官方试点、采购评估、授权试用流程 |
| 共享 key | 明显高风险 | 客户自有 key、权限隔离、BYOK 管理 |
| 逆向接口 | 明显高风险 | 官方 API、授权 SDK、合规接入 |
| 不排队、满血 | 容易暗示规避限制 | SLA、错误率监控、容量规划 |
| 普通人副业 | 容易包装套利机会 | 中小团队 API 采购入门 |
| 有需要联系 | 交易导向强 | 领取检查表、预约合规诊断 |

## 3. 标题改写示例

### 示例 1

高风险标题：

> 普通人怎么卖 token，兼职也能做。

合规改写：

> 中小团队如何理解 AI API 采购、调用成本和合规接入？

### 示例 2

高风险标题：

> 低价 API 中转站，测试账号有需要联系。

合规改写：

> 企业 API 采购前，先做这 5 项供应商风险评估。

### 示例 3

高风险标题：

> Claude / Codex 倍率表，低价稳定。

合规改写：

> 同样使用多模型 API，为什么两家公司账单能差一倍？

### 示例 4

高风险标题：

> 满血不排队，国内也能用。

合规改写：

> 企业评估模型 API 稳定性时，SLA、错误率和容量要怎么看？

### 示例 5

高风险标题：

> API 充值怎么做，普通人入局第二期。

合规改写：

> 企业从试用 AI 到正式采购，中间要过哪些流程？

## 4. 承接方式改写

| 原承接方式 | 风险 | 合规承接 |
|---|---|---|
| 私信拿低价通道 | 交易导流 | 评论领取 API 采购风险自查表 |
| 加群拿测试账号 | 账号/key 分发风险 | 预约官方试点和采购评估流程咨询 |
| 发送倍率表 | 价格套利导向 | 发送总拥有成本测算表 |
| 联系充值 | 代充风险 | 预约预算治理和账单透明化诊断 |
| 推荐中转站 | 反向代理转售风险 | 供应商尽调清单和 SLA 对比表 |

## 5. 安全内容结构

推荐结构：

1. 开头：一个真实企业痛点。
2. 正文：拆 3 到 5 个风险或判断标准。
3. 画面：白板、表格、流程图、检查清单。
4. 结尾：引导领取合规资料或预约诊断。
5. 承接：采购清单、成本测算表、审计模板、路由方案。

示例：

> 开头：很多团队第一个月 API 账单还能接受，第三个月突然失控。
>
> 正文：不是模型一定贵，而是没有做任务分层、模型路由、预算阈值和异常调用审计。
>
> 结尾：我整理了一张 API 成本治理检查表，可以用来先看你们的问题出在哪里。

## 6. 禁止包装式改写

以下改写不合格：

- 把中转站改名叫网关，但继续做共享 key 或转售。
- 把测试账号改名叫试用账号，但继续分发账号。
- 把低价倍率改名叫成本优势，但继续承接低价通道。
- 把私信拿资源改名叫咨询，但实际仍交付高风险资源。
- 用企业采购话术规避平台审核，但业务实质不变。

合格改写必须同时改变三件事：

1. 内容表达。
2. 承接资料。
3. 真实交付方式。
